# PactSpec Governance

PactSpec is stewarded by a founding steward to keep momentum while adoption is early.

## Stewardship
Founding steward: Dave, project owner.
The steward maintains the spec, coordinates releases, and accepts contributions.

## Decision Making
We use an RFC process and aim for consensus.
If consensus is not reached, the steward makes the final decision.
Any change that affects wire compatibility or `specVersion` requires an RFC.

## Path to a Multi-Vendor Working Group
When PactSpec reaches 3 or more real adopters, the steward will initiate a transition to a multi-vendor working group.
An adopter is a production system that publishes PactSpec or consumes it in production.

## Scope
This governance applies to the PactSpec specification and schema.
Reference implementations and the registry are non-normative.

## Adopters
Production adopters are listed in `ADOPTERS.md`.

## Code of Conduct
We follow the Contributor Covenant, version 2.1.
